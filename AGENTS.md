# 项目协作规则

- 所有回复一律使用中文。
- 修改代码中已有的注释（包括函数注释、行内注释、文件头注释）时，必须逐条列出拟修改的内容并征求用户确认，确认后才能写入文件。
- 检索或阅读代码时发现已有注释与当前实现不一致、语义过期或存在疑问时，必须主动列出问题注释并与用户讨论，确认后再修改。
- 新增注释无需确认，可直接写入。
- 修改代码前，先简要说明将要修改的文件和目的。
- 优先遵循项目现有代码风格、目录结构和命名习惯。
- 不要擅自删除、回滚或覆盖用户已有改动。
- 不要执行破坏性命令，例如 `git reset --hard`、强制删除文件或大范围重写历史，除非用户明确要求。
- 涉及依赖安装、联网下载、权限提升或可能影响本机环境的操作时，先征求用户确认。
